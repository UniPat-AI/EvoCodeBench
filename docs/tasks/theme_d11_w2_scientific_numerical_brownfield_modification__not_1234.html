Difficulty analysis

This task implements nlsolve, a sparse-matrix iterative solver toolkit in C: read a linear system Ax = b from files, solve it with CG/GMRES plus preconditioners, analyze matrix structure, and report convergence — 5 rounds. Writing this in C (manual memory, exact stdout/stderr discipline) raises the bar on interface details rather than the math itself.

Four requirements defeat nearly the whole field across 12 models. true-residual (failed by 9 models) requires the residual to match an independently computed reference value; matrix-analysis (8 models) expects exactly 7 nonzeros reported for an expanded tridiagonal; error-handling (8 models) demands that generating a Laplacian without an output file exits with code 1; and duplicate-entry-merge (8 models) requires CG with IC(0) to converge correctly when the input matrix contains duplicate diagonal entries. Also commonly missed: ic0-fallback-partial (6 models), where a zero-diagonal IC(0) fallback must report "preconditioner: none" — all precise numerical or output contracts that case-level scores alone cannot capture.

Performance gap

The striking pattern is how flat the round scores are against how high the case scores run. GPT-5.5 and GLM-5.2 share the lead at 2/5 rounds — GPT-5.5 at 98% case pass and GLM-5.2 at a perfect 100% — both halted by the same cluster of residual-verification and matrix-analysis contracts. DeepSeek-V4-Pro reaches 1/5 at 85% case pass, losing ground on the more demanding preconditioner and convergence rounds.

The remaining nine models are 0/5 rounds, yet the spread is wide: MiniMax-M3 (99%), Kimi-K2.6 (98%), Qwen3.7-Max (94%), and Opus-4.8-xhigh (95%) are functionally close to the leaders on cases but fall on the all-or-nothing round gate — mostly tripped by true-residual, duplicate-entry-merge, and error-handling. DeepSeek-V4-Flash (75%), Qwen3.6-Plus (74%), and GLM-5.1 (70%) sit in a mid-tier that also misses matrix-analysis. Only Kimi-K2.7-Code (37%) and MiniMax-M2.7 (23%) genuinely struggle at the case level, the latter failing broadly across true-residual and duplicate-entry-merge.
