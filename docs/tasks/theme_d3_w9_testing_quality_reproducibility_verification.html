Difficulty analysis

This task implements repro-verify from a written spec (/app/docs/spec_v1.md): a CLI that verifies computational reproducibility — running deterministic checks, detecting nondeterminism, normalizing environments, matching against a reference, and producing summary/diff/graph reports over a manifest of steps. 7 rounds. The spec-driven nature means the exact command surface and JSON output shapes are fixed, and the round verifier is all-or-nothing.

The commonly-failed requirements span the core verification logic and its reporting: report-summary (8/10), then verify-deterministic, nondeterminism-detect, env-normalization, cross-pass-consistency and reference-match (7/10 each). Failure reasons cluster into two modes — wrong exit codes and null output. verify-deterministic repeatedly exits 2 where exit 0 was expected; nondeterminism-detect exits 2 instead of 1; env-normalization reports "status:ERROR" instead of PASS; and cross-pass-consistency / reference-match return "got null" where booleans were required. dependency-analysis returning an empty "got " (no topological order) and summary-subcommand exiting 2 break the weaker models earlier.

Performance gap

The ranking here is inverted from the frontier norm. Qwen3.6-Plus solves it cleanly: 7/7 rounds, 100% case pass — the sole perfect run. GPT-5.5 is next at 2/7 (89%). Then the surprise: every other model, including Opus-4.8, scores 0/7.

Case pass reveals how misleading those zeros are. Kimi-K2.6 passes 98% of cases at 0/7, missing only report-summary and cycle-detection — a virtually complete implementation that the binary reward scores identically to a real failure. Opus-4.8 sits at 76% (0/7), blocked by the verify-deterministic / nondeterminism / env-normalization exit-code cluster. Below them the case rates collapse genuinely: GLM-5.1 65%, Qwen3.7-Max 70%, MiniMax-M3 42%, DeepSeek-V4-Pro 27%, and DeepSeek-V4-Flash just 7%. MiniMax-M2.7 also hits build✗ in 4 rounds (its own code failing to compile). The headline lesson is the Kimi/Opus pair: two strong models at 98% and 76% cases scored the same 0/7 as models that barely run, purely because they each tripped a couple of spec-exact requirements.
