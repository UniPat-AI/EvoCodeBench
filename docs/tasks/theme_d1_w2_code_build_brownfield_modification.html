Difficulty analysis

This is a hard brownfield task: minicc is an existing bytecode compiler and stack-based VM for a small language called Mini, and the agent must evolve it — adding language features like for-in/for-of loops, closures, exceptions and hashmaps to a codebase whose lexer, parser, codegen and VM already have fixed contracts. 8 rounds. Brownfield raises the bar: new features must thread through every layer of the existing pipeline without breaking what already passes.

The commonly-failed requirements are real language semantics, missed by 9-10 of 10 models: for-of-values and try-catch (10/10), for-in-indices, try-catch-control, combined-features and capture-by-ref (9/10). The failure reasons are crisp wrong answers from the VM: capture-by-ref yielding "actual: 10" where 20 was expected (closures capturing by value instead of reference), combined-features printing "60" instead of "0 1 2", short-circuit emitting "bad false" instead of "0", and loops producing empty output. existing-programs regressions ("actual: 40 45 225" vs "30 15 300") show models breaking the pre-existing test suite while adding features.

Performance gap

This task produces the steepest single-model lead in the set. Opus-4.8 scores 6/8 rounds at 98% case pass, losing only a handful of edge cases (for-in-indices, try-catch-control, hashmap-literal — 1 case each). No other model clears more than 2 rounds: GPT-5.5 manages 2/8 (52%) and Kimi-K2.6 1/8 (71%).

Everyone else lands at 0/8 rounds, and the case rates fan out widely — MiniMax-M3 holds 84% but never a clean round, while DeepSeek-V4-Flash drops to 20% and Qwen3.7-Max to 25%. Three models (Qwen3.6-Plus, GLM-5.1, DeepSeek-V4-Flash) fail to build their own compiler at least one round (1build✗) — the reference oracle compiles fine, so this is the agent's own code not compiling, the brownfield trap of editing a real codebase. The separation here is almost entirely Opus vs. everyone: only Opus gets closures and capture-by-reference semantics right across rounds; the rest get the happy path but break on the stateful, scoping-sensitive constructs.
