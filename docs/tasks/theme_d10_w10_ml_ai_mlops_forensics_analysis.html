Difficulty analysis

This task implements mlforge, a CLI for ML-checkpoint forensics: ingest checkpoints, extract metadata, reconstruct lineage and timeline, diff versions, query, and detect anomalies (gradient spikes, severity levels) — 9 rounds, 59 → 111 cumulative cases. Unlike d7_w5, the work is mostly pure data transformation, so the chain never aborts: all 10 models run to round 9. That makes it a clean capability probe.

The single load-bearing requirement is ingest (failed by 6 models, with reasons like "got null"): the timeline, lineage, and query features all read from ingested state, so a wrong ingest cascades downstream into timeline, timeline-conflict, timeline-gap-threshold and metadata failures. error-handling (exiting 0 when it should fail) is the other shared miss.

Performance gap

This task discriminates almost binarily. Opus-4.8 and MiniMax-M3 pass 100% of cases on every round — full marks. Just below them, Kimi-K2.6 (98% cases) and GPT-5.5 (97%) each lose to one narrow requirement — Kimi to detect-checkpoint-field/severity-5level, GPT-5.5 to lineage (15 cases) — enough to drop their round reward to 2/9 and 1/9 but reflecting near-complete implementations.

The remaining six models (DeepSeek-V4-Pro, the Qwen pair, GLM-5.1, the DeepSeek/MiniMax fast tiers) sit at 74–85% case pass but 0/9 rounds, and their failures all trace back to the ingest/metadata foundation cascading into timeline and lineage. The gap here is not breadth of effort — everyone runs all 9 rounds — but whether the model got the data-ingestion contract exactly right. MiniMax-M3 punching up to a perfect score alongside Opus is the standout result.
