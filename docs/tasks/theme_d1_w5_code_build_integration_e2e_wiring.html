Difficulty analysis

This task wires up buildctl, a Go CLI that orchestrates multi-target build pipelines: dependency resolution, parallel execution, caching, hooks, variable substitution and config imports. 8 rounds. As an integration / e2e-wiring task, the difficulty is making independently-correct pieces work together — affected-set computation feeding the planner, hooks chaining in order, variables resolving with the right precedence across imported files.

The hardest commonly-failed area is incremental dependency analysis: deps-affected and deps-affected-plan top the fail lists with enormous case counts (108 and 139 for DeepSeek-V4-Pro), usually as a hard error — "expected exit=0, got exit=1" — meaning the subcommand crashes rather than computes the wrong set. The wiring-specific misses are equally telling: hook-chain running "main" before "pre1" (hooks fire in the wrong order), imports-transitive returning "top" instead of "foundation" (transitive imports not followed), profile-critical-path returning an empty path where it should exclude a failed target, and cache-key-fix producing empty keys ("a= b="). The shared core profile-critical-path, cache-key-fix, fail-fast and the retry-hook requirements are missed by 8-9 of 10 models.

Performance gap

This task is a great leveller — the ceiling is just 2/8 rounds, shared by MiniMax-M3, DeepSeek-V4-Pro and Qwen3.6-Plus. But round score badly misranks capability here: Opus-4.8 and GPT-5.5 sit at only 1/8 rounds yet pass 95% of cases, with their losses concentrated almost entirely in one cluster — imports, imports-transitive, variables-importpath and variables-precedence. They build a near-complete orchestrator and miss a clean round only because import/variable resolution is off.

The genuinely weak models fail far more broadly and at the structural level: Kimi-K2.6 (83%), DeepSeek-V4-Flash (71%), GLM-5.1 (67%) and MiniMax-M2.7 (66%) drown in deps-affected-plan/deps-affected crashes (100+ cases each). Four models hit build✗ at least one round (MiniMax-M3, DeepSeek-V4-Pro, DeepSeek-V4-Flash, GLM-5.1) — their own code failing to compile, not a task defect. So the real divide is Opus/GPT-5.5 (everything works but imports) vs. the rest (the affected-set engine never runs), even though the round table compresses both into the same 1-2 band.
