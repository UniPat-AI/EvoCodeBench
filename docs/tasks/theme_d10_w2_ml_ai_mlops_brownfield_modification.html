Difficulty analysis

This is a brownfield task: extend the existing nexus-ml-pipeline engine so it runs complete DAG workflows (steps with dependencies, conditions, retries, captured output, artifacts), 8 rounds, 32 → ~75 cumulative cases. Brownfield is harder than greenfield because the agent must respect the existing code's contracts, not just write fresh code.

What makes this task stand out is a universal core of failures — requirements missed by all 10 models: variable-defaults, step-env, unresolved-var and retry-history (10/10), plus train-integration, execution-timeout, evaluate-step and artifacts (9/10). The failure reasons pinpoint the contract gaps: variable substitution producing empty output ("status:succeeded out="), the train step not registering a tracked experiment ("no experiment_id"), retry metadata fields missing ("len= att= err="), and errors swallowed instead of surfaced ("status: error="). These are precise interface details that even the strongest models miss.

Performance gap

Two clusters, with the widest case spread of any task. The strong cluster — Opus-4.8, GPT-5.5, MiniMax-M3 at 76–80% case pass — implements the pipeline structure and only loses the subtle variable-resolution and retry-history semantics above. The weak cluster falls steeply: Kimi-K2.6 55%, DeepSeek-V4-Pro 49%, GLM-5.1 40%, DeepSeek-V4-Flash 36%, Qwen3.7-Max 32%, MiniMax-M2.7 26%.

The weak models fail on a different, more basic layer — allow-failure, capture-output, dry-run, condition-logic and step-status-ref — i.e. the control-flow and output-plumbing of the DAG itself, not just the fine-grained variable semantics. So the gap is layered: strong models get the plumbing and lose only the edge-case contracts; weak models never get the plumbing working. MiniMax-M2.7 also aborts the chain after round 2, the only model that fails to run to completion here.
