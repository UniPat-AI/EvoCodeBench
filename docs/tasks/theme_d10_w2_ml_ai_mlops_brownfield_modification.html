Difficulty analysis

This is a brownfield task: extend the existing nexus-ml-pipeline engine so it runs complete DAG workflows (steps with dependencies, conditions, retries, captured output, artifacts), 8 rounds, 32 → ~75 cumulative cases. Brownfield is harder than greenfield because the agent must respect the existing code's contracts, not just write fresh code.

What makes this task stand out is a near-universal core of failures across 12 models — requirements missed by nearly every model: dry-run (11/12), capture-output, variable-defaults, step-env, variables-map and allow-failure (each 10/12). The failure reasons pinpoint the contract gaps: dry-run mode not suppressing execution ("cmd="), captured output fields absent or empty ("status: var="), variable substitution producing empty output ("status: out="), and allow-failure not suppressing pipeline abort ("out="). These are precise interface details that even the strongest models miss.

Performance gap

The gap is extreme, with the widest case spread in the suite (5–100%). Only Opus-4.8-xhigh completes all 8 rounds at 100% case pass; GLM-5.2 scrapes 1 round at 40% case. The strong partial cluster — Qwen3.6-Plus 87%, GPT-5.5 84%, MiniMax-M3 67% — builds the pipeline structure correctly but loses the fine-grained variable-resolution and dry-run semantics above. The weak cluster falls steeply: Qwen3.7-Max 50%, DeepSeek-V4-Pro 49%, DeepSeek-V4-Flash 46%, Kimi-K2.6 44%, GLM-5.1 and GLM-5.2 40%, Kimi-K2.7-Code 25%, MiniMax-M2.7 5%.

The weak models fail on a more basic layer — capture-output, step-env and allow-failure — i.e. the control-flow and output-plumbing of the DAG itself, not only the edge-case variable contracts. MiniMax-M2.7 collapses after the first round at 5% case pass, the only model that fails to make meaningful progress across the chain.

Is the task fair? (top-model check)

Claude Opus-4.8 solved all 8 rounds at full marks (8/8 rounds, 100% case pass every round), confirming the task and its test suite are sound: a capable agent can satisfy every requirement, including the notoriously missed dry-run, capture-output, and allow-failure contracts. Failures by other models therefore reflect genuine capability gaps — particularly in tracking fine-grained variable-resolution and execution-suppression semantics across a multi-round brownfield chain — not defects in the task design or grading.
