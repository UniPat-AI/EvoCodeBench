Difficulty analysis

This task stands up flowctl, a CLI that reads a YAML description of a data pipeline and actually runs it end-to-end: chained stages (group, pivot/unpivot, window, bucket, rank, join) over CSV/NDJSON, 9 rounds growing from a handful to dozens of cumulative cases. The work is integration wiring — each stage must compose with the next, and the config parser must accept or reject every stage's options exactly.

The hardest requirements, failed by 8 of 10 models, are the stage implementations themselves: unpivot-config (28 cases), window-aggregates (26), unpivot-axes-config (22), window-config (14) and group-aggregates. The failure reasons show the weak models never built these stages — the parser rejects them outright with "unknown stage type: group", "unknown stage type: pivot", and "window: session is not yet supported". Even the strong models trip on edge semantics: late-data handling in window-late ("strict: late row 3") and abort-on-bad-config in unpivot-config ("c31 should abort").

Performance gap

The separation here is unusually stark. GPT-5.5 (6/9, 99% cases) and Opus-4.8 (5/9, 99%) genuinely implement the whole pipeline and lose only a few edge cases — GPT-5.5 on window-empty-windows and window-aggregates, Opus on unpivot-config and bucket-strict. They are the only models that wired the system together.

Everyone else collapses. Kimi-K2.6 manages 1/9 at 73% cases but still leaves group and pivot as unrecognized stage types. The remaining seven — MiniMax-M3 (22%), DeepSeek-V4-Pro (12%), and then GLM-5.1, Qwen3.6-Plus, Qwen3.7-Max, DeepSeek-V4-Flash and MiniMax-M2.7 all at 1–17% case pass and 0/9 rounds — fail an almost identical block of requirements (unpivot-config 28, window-aggregates 26, unpivot-axes-config 22…), meaning they never implemented the core stages. This is not the binary-reward artifact seen elsewhere: here the low round scores reflect genuinely low case pass. The task cleanly divides models that can build a composable pipeline from those that produce a parser that rejects most of the spec.
