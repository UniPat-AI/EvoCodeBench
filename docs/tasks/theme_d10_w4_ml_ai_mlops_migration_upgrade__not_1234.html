Difficulty analysis

This is a long migration task on mlpipe, a Go CLI for ML inference-pipeline management: a model registry, checkpoint loading, comparison, pruning, ensembles, import/export and explain — 13 rounds, 47 → 70 cumulative cases. The unusual depth means a model has many chances to accumulate features but also many places for one broken subcommand to surface.

The commonly-failed requirements are mostly whole subcommands that exit non-zero: explain (8 models — "exit=1 expected=0"), compare (7), prune (6), prune-profile (6), diff (6 — "mismatch") and compat (6). For the strong models the single recurring gap is import (failing with "exit=1 expected=0"), which several otherwise-complete implementations cannot satisfy.

Performance gap

The top tier is genuinely strong here. Opus-4.8 passes 100% of cases and clears 11/13 rounds, losing only upgrade (2 cases). Kimi-K2.6 (98%, 9/13) and MiniMax-M3 (98%, 8/13) both fall almost entirely to import (6 cases each) plus a stray clone/explain case — near-complete tools held back by one subcommand. GPT-5.5 drops further to 85%/5-of-13, its misses spreading into compare (41 cases), explain (30) and prune (23).

The weak tier scores 0–4/13 but most still pass high case rates, the binary reward hiding the spread: DeepSeek-V4-Pro (93%, 4/13), DeepSeek-V4-Flash (92%, 0/13) and Qwen3.6-Plus (91%, 0/13) all sit above 90% cases yet land near zero rounds, failing scattered subcommands like ensemble-info, info-version and batch. GLM-5.1 (78%) and Qwen3.7-Max (75%) are genuinely weaker, losing diff (32 cases each), stats and validate-input broadly. DeepSeek-V4-Flash's 92% case pass at 0/13 rounds is the starkest reward/capability mismatch in this task.
