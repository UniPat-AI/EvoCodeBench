Difficulty analysis

This task builds dpipe, a deterministic Go data pipeline: ingest structured data, then validate, profile, transform (derive, fill, rollup, normalize, temporal, pivot…), compare, compute lineage, emit a reproducible manifest with a stable content hash, and audit/verify the run — 15 rounds, the longest chain in the suite, accumulating hundreds of cumulative cases. Determinism is the crux: the manifest must reproduce an exact checksum ("expected=a7b884ca… actual="), so any nondeterministic ordering or formatting fails the round.

The commonly-failed requirements span the whole pipeline — manifest, profile, pipeline, lineage, error-handling and transform.normalize are each missed by all 10 models. Because the suite is cumulative over 15 rounds, a single early requirement a weak model never implements (e.g. profile or transform.derive) re-fails in every later round, which is why the weak side's per-requirement counts run into the hundreds.

Performance gap

This is a marathon where everyone stumbles but the gap is in how much working code is hidden under a 0. The top tier is Opus-4.8 (4/15, 93% cases) and Qwen3.7-Max (0/15 but 89% cases) — Qwen3.7-Max passes nearly nine of ten cases across fifteen rounds yet scores zero rounds, the sharpest example here of the binary reward masking near-complete work. GPT-5.5 also reaches 4/15 at 85%.

Below them the case pass falls off and reflects real breakage: MiniMax-M3 75%, Kimi-K2.6 72%, Qwen3.6-Plus 65%, DeepSeek-V4-Pro 41%, DeepSeek-V4-Flash 34%, GLM-5.1 19% and MiniMax-M2.7 just 6%. The bottom models' huge per-requirement counts (GLM-5.1 transform.derive 819, MiniMax-M2.7 profile 1071) are the same unimplemented requirement re-failing across all 15 cumulative rounds, not 800+ distinct bugs — every model here ran all 15 rounds (no build failures), so the lows are genuine missing functionality. So the real story is two-layered: at the top, models that mostly work but cannot lock the deterministic manifest across all 15 rounds; at the bottom, models that never implemented whole pipeline stages, re-failing them round after round.
