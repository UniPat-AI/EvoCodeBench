Difficulty analysis

This task implements the ml-eval-report-engine from a written spec: evaluate classification and regression results, compute bootstrap confidence intervals, detect direction-aware regressions, and emit summary/compare subcommands as JSON — 7 rounds of progressively expanded requirements. The work is spec-driven data transformation, so all 12 models run every round; the chain never aborts, making this a clean capability probe.

The most commonly failed requirements are the statistical and CLI-surface pieces: direction-aware-regression (8 models — the verifier expects a canonical label, e.g. expected='lower-is-better' actual='lower_is_better'), bootstrap-ci (7), binary-eval (7 — expected=0.7 actual=0), summary-subcommand (6), multiclass-eval (6 — cannot parse actual=null), and regression-detection (6 — expected exit=1 actual=0). Many of these are whole-subcommand failures rather than numeric precision misses — the evaluators produce wrong types or crash entirely.

Performance gap

The task splits into a sharp top tier and a broad failure band. Opus-4.8 and MiniMax-M3 each pass all 7 rounds at 100% case rate — the only perfect scores. Kimi-K2.7-Code (87%, 6/7) drops one round, most likely to direction-aware-regression. GPT-5.5 (97%, 2/7) and GLM-5.2 (98%, 2/7) hold very high case rates but cannot clear the full round gate on 5 rounds each, suggesting narrow but load-bearing misses on direction-aware-regression or binary-eval.

The remaining seven models land at 0/7 or 1/7: DeepSeek-V4-Flash (1/7, 88%), DeepSeek-V4-Pro (1/7, 76%), and Qwen3.7-Max (1/7, 73%) each eke out a single passing round before stalling, while Qwen3.6-Plus (66%), GLM-5.1 (56%), MiniMax-M2.7 (44%), and Kimi-K2.6 (42%) fail broadly across bootstrap-ci, binary-eval, multiclass-eval, and summary-subcommand — the core evaluators themselves are either absent or produce wrong output. The binary round gate flattens a genuine spread from 88% down to 42% into a uniform 0/7 for most of the field, hiding substantial differences in working capability.
