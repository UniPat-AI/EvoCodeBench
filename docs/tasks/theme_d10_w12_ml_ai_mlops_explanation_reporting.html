Difficulty analysis

This task implements the ml-eval-report-engine from a written spec: evaluate classification and regression results, compute bootstrap confidence intervals, detect direction-aware regressions, and emit summary/compare subcommands as JSON — 7 rounds, 68 → 78 cumulative cases. The work is spec-driven data transformation, so all 10 models run every round; the chain never aborts, making this a clean capability probe.

The commonly-failed requirements are the statistical and CLI-surface pieces: direction-aware-regression (6 models — the verifier expects a labelled direction, e.g. expected='lower-is-better' actual='null'), compare-exclude-metric (6), binary-eval (5), multiclass-eval (5), regression-eval (5) and bootstrap-ci. Several misses are whole-subcommand crashes rather than wrong numbers — summary-subcommand and summary-json fail with "expected exit=0 actual=2", i.e. the command errors out entirely.

Performance gap

This task separates sharply at the top. Opus-4.8 and Kimi-K2.6 each pass 100% of cases on all 7 rounds — perfect scores. MiniMax-M3 (99%) loses a single round to one ordering detail, regression-analysis-order (3 cases), landing 6/7. Below them GPT-5.5 (76%, 3/7) and Qwen3.7-Max (92%, 2/7) each fall to one or two subcommands — GPT-5.5 to summary-json/compare-exclude-metric, Qwen3.7-Max almost entirely to direction-aware-regression (29 cases).

The remaining five models all score 0/7 but spread wide on cases: DeepSeek-V4-Flash holds 73% and GLM-5.1 56%, while DeepSeek-V4-Pro (23%), MiniMax-M2.7 (22%) and Qwen3.6-Plus (30%) fail broadly across binary-eval, bootstrap-ci, multiclass-eval and summary-subcommand — the core evaluators themselves crash. The binary round reward flattens DeepSeek-V4-Flash's genuine 73% into the same 0/7 as MiniMax-M2.7's 22%, hiding a real three-fold difference in working capability.
