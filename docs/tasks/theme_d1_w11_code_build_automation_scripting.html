Difficulty analysis

This is a brownfield build-automation task: extend forgekit, a bash-based build orchestrator for a monorepo of eight C packages (four libraries, four services), so it does dependency-aware incremental builds — content-hash caching, mtime tracking, selective rebuilds, impact analysis, and profile-scoped clean — across 8 rounds. The agent must reproduce subtle filesystem and caching semantics in shell.

One requirement dominates everything: selective-rebuild, failed by all 10 models. The contract is that touching one library must propagate a rebuild to its dependents — "--changed after libbase touch updates libcrypto artifact", i.e. the dependent's artifact mtime must advance — and no model gets the dependency-propagation right. The related cache contracts cache-manifest-mtime and content-hash-cache (10/10 each) fail the same way: a manifest touch or Makefile content change must bump the downstream artifact mtime, and it doesn't. internal-suite (9/10, "internal test suite fails") and profile-scoped-clean / single-pkg-clean-cache (8 each) round out the universal core.

Performance gap

Because selective-rebuild blocks everyone, the round scores compress into a narrow 0–2/8 band that hides large case-level differences. The two 2/8 leaders could not be more different: GLM-5.1 reaches 2/8 at 93% case pass — a strong implementation losing only the rebuild/cache propagation — while MiniMax-M3 also reaches 2/8 but at just 60%, with selective-rebuild blowing up to 146 cases and impact-analysis (56) and deps-why-path (23) also broken. Equal rounds, wildly unequal capability.

The high-case-pass cluster — Opus-4.8 (95%), Qwen3.6-Plus (92%), DeepSeek-V4-Pro and Qwen3.7-Max (90%), Kimi-K2.6 (88%) — all score only 1/8, their misses confined almost entirely to selective-rebuild (45–58 cases) plus a few cache-manifest-mtime / content-hash-cache / internal-suite cases. They built nearly the whole orchestrator and are penalized to a single round by one dependency-propagation bug. At the bottom, DeepSeek-V4-Flash (69%, 0/8) and MiniMax-M2.7 (79%, 0/8) additionally miss profile-build-dirs and produce broader breakage. The round column makes a 60%-vs-95% gap look like a one-round difference.
