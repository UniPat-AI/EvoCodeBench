Difficulty analysis

This task builds flowr, a greenfield CLI workflow orchestrator: read a YAML file describing a DAG of shell commands and run them in dependency order with bounded parallelism, retries, conditional steps, and dry-run, across 7 rounds. It is broad feature-plumbing work where finishing the whole feature set matters more than getting one numerical contract exactly right.

The widely-failed requirements are all later-round features: F15 (null cell values), F13 (step status and shape), the regression guard (attachment/error-count/duration shape), F23 (3-D box rendering), F27 (retry-count field), and F28 (artifact null handling) — each failed by 10 of the 12 models. Building the DAG core is achievable across the field; layering these later-round data-shape and surface contracts on top within the session is what 12 models cannot reach.

Performance gap

The hierarchy holds, but the bar is high across all 12 models. Opus-4.8 leads at 2/7 rounds (84% cases), accumulating the highest case coverage in the field before the heavier later features stop it. GPT-5.5 completes 1/7 rounds (60% cases), building breadth but falling short of whole-round thresholds as complexity compounds. Qwen3.7-Max shows the same partial-build shape without finishing any round (0/7, 49%), followed by MiniMax-M3 (0/7, 35%), DeepSeek-V4-Flash (0/7, 30%), and MiniMax-M2.7 (0/7, 29%).

The lower tier implements only a fraction of the command surface: Kimi-K2.6 (0/7, 24%), DeepSeek-V4-Pro (0/7, 21%), GLM-5.1 (0/7, 12%), and at the bottom Kimi-K2.7-Code, Qwen3.6-Plus, and GLM-5.2 (0/7, 3%, 0%, 0%). The round score and the case score agree: this is a hard, honest from-scratch build where only the two strongest models land any completed rounds at all.
