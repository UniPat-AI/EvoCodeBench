Difficulty analysis

This is a migration/modernization task on gomlops, a monolithic Go ML-lifecycle HTTP service: modernize it into experiment tracking, a model registry, pipeline execution and batch inference — 7 rounds across 12 models. As a brownfield migration the agent must preserve existing API contracts while reworking internals.

The commonly-failed requirements cluster in the pipeline and inference engine: pipeline-validation and batch-inference each trip 10 of 12 models, followed by regression (9 models — "expected value matches expectation, got (empty)"), and version-comparison, trimmed-mean, and model-lineage (7 models each). These are precise stateful contracts, not surface plumbing.

Performance gap

Opus-4.8 stands alone at 100% case pass, 7/7 rounds — the only model with no listed failures. A narrow second tier clears a handful of rounds at high case rates: MiniMax-M3 (99%, 4/7) and Qwen3.7-Max (83%, 2/7) each fall short on pipeline-validation and batch-inference; GPT-5.5 (89%, 1/7), Qwen3.6-Plus (97%, 1/7), Kimi-K2.6 (79%, 1/7), Kimi-K2.7-Code (73%, 1/7), and DeepSeek-V4-Flash (64%, 1/7) all clear exactly one round. Their case rates against a single cleared round illustrate the binary-reward gap — near-complete implementations scored as mostly-failing.

Three models score 0/7 rounds despite non-trivial case pass: DeepSeek-V4-Pro (79%), GLM-5.1 (83%), and MiniMax-M2.7 (79%) each fail broadly on pipeline-validation, batch-inference, and model-lineage. GLM-5.2 clears 3/7 rounds at only 69% cases — high round count masking shallow per-case coverage. The breadth of regression and trimmed-mean failures across 7–9 models confirms that numerical contract correctness, not code structure, is the decisive separator.

Is the task fair? (top-model check)

Claude Opus-4.8 solved all 7 rounds at 100% case pass, including every round that defeated 10 of 12 other models — confirming that pipeline-validation, batch-inference, regression, and trimmed-mean are correctly specified and fairly graded. Weaker models' failures on these requirements reflect genuine capability gaps in stateful brownfield migration, not defects in the task or its test suite.
