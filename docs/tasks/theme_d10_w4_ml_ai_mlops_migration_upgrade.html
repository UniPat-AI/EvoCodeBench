Difficulty analysis

This is a migration/modernization task on gomlops, a monolithic Go ML-lifecycle HTTP service: modernize it into experiment tracking, a model registry, pipeline execution and batch inference — 7 rounds, 67 → 124 cumulative cases. As a brownfield migration the agent must preserve existing API contracts while reworking internals.

The commonly-failed requirements cluster in the pipeline engine: pipeline-step-retry (7 models) and pipeline-validation (7 — "expected value matches expectation, got 1") lead, followed by batch-inference (5 — failures return null), pipeline-step-timeout (5), normalize-symmetric and regression. The experiment-tracking edges — experiment-run-update and experiment-mutation-effects — also recur, where a mutation should change tracked state but the read-back returns null. These are precise stateful contracts, not surface plumbing.

Performance gap

Opus-4.8 stands alone at 100% case pass, 7/7 rounds — the only model with no listed failures. A tight near-strong tier sits just behind at very high case rates but far fewer rounds: MiniMax-M3 (97%, 4/7) losing experiment-metric-history and experiment-run-search; GPT-5.5 (99%, 3/7) and Kimi-K2.6 (96%, 3/7) each tripping on a handful of retry and update cases. Their 96–99% case pass against 3–4/7 rounds is the binary-reward gap in miniature — near-complete implementations scored as mostly-failing.

The weaker tier still posts respectable case rates but compiles unreliably or misses broadly. DeepSeek-V4-Pro (97%) and GLM-5.1 (80%) each hit build✗ rounds where their own code failed to compile; Qwen3.7-Max (83%, 1/7) and Qwen3.6-Plus (91%, 1/7) lose wider sets — model-lineage, batch-inference, output-transforms. MiniMax-M2.7 holds 93% cases yet scores 0/7, failing model-version-filter and model-annotations. DeepSeek-V4-Flash is the genuine outlier at 59% with a build✗ round and broad regression/experiment-leaderboard failures.
