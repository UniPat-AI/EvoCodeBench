Difficulty analysis

This is a security-hardening task: build a complete ML model-serving platform as one Go binary — an HTTP gateway with authentication, role-based authorization, audit logging, input validation and request-size limits guarding model create/upload/deploy/inference — 8 rounds, ~135 cumulative cases. Unlike the pure data-transformation tasks, much of the difficulty is getting the auth and validation middleware exactly right, because every protected endpoint depends on it.

The commonly-failed requirements are the security primitives themselves: model-create (8 models), auth-register (7), password-hash (5), auth-required (5), body-size-limit (5) and model-name-validation (5). The failures show two distinct break modes — a broken auth layer rejecting valid requests ("expected status 201, got 401 (register deployer)") versus a too-permissive one ("expected status 400/409/422, got 201 (duplicate model)") — plus audit gaps where sanitized details come back empty ("Expected truncated details (1-280 chars), got length 0").

Performance gap

This task inverts the usual ranking. The strong tier is the lighter models: Qwen3.6-Plus leads at 6/8 (97% cases), Kimi-K2.6 at 5/8 (96%), and DeepSeek-V4-Pro and DeepSeek-V4-Flash both at 4/8 (96%/94%) — they get the auth and authorization middleware working and lose only narrow edges like audit-sanitization (2 cases).

The usual front-runners crater. Opus-4.8 and GPT-5.5 both score 0/8 at 56% and 63% case pass, each with multiple build✗ rounds where their own code failed to compile, and the rounds that did run fail in a 401 cascade — auth-register, model-create and body-size-limit all report "got 401", a single broken auth gate failing everything behind it. Qwen3.7-Max shows the identical 63%/0-of-8 pattern. MiniMax-M3 runs all rounds at 81% but its registration validation returns 400 on valid names ("expected status 201, got 400"). At the bottom, MiniMax-M2.7 (13%, 3 build✗) and GLM-5.1 — which fails to compile on all 8 rounds for 0% cases — never get off the ground. Here the binary reward and the model ranking both flip: hardening middleware, not algorithmic depth, is what the frontier models stumbled on.
